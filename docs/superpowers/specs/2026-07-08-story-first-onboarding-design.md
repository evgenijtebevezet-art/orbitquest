# OrbitQuest — story-first онбординг: персонажи, новый пролог, упрощение, моушн

Фидбек игрока №1: «слишком непонятно, UI сложный, надо разжёвывать» + «сюжет не считывается».
Утверждено в чате (владелец): четыре направления одной пачкой.

## 1. Новый текст пролога (утверждён дословно в чате)

7 сцен переписаны простым языком: «кто мы» (1–3), «зачем мы» (4–6), «почему именно ты» (3, 7).
Финальные формулировки — в переписке 2026-07-08, вшиваются в `content/prologue/scenes.json`
без изменения схемы (id/структура сцен не меняются, только paragraphs/actionLabel).
Правки канона тона: разговорный взрослый регистр допустим («если что», «протухают, как еда
без холодильника»), PIX сохраняет технический юмор.

## 2. Персонажи-портреты (codex-image, консистентность по референсам)

- Генерируются портреты: **KORA** (световое ядро с кольцами), **PIX** (округлый дрон с
  экраном-лицом), **VEGA** (человек, диспетчер, портрет+waveform). Референсы — существующие
  сцены (`scene-3`, `scene-6`, `scene-5`) + стайлборд №6; стиль тот же.
- Формат: квадратные портреты, тёмный/прозрачный фон, вес после webp ≤120KB каждый.
- **Всплывающий говорящий:** в прологе портрет активного спикера появляется у плашки
  (Motion slide-in + лёгкий bounce). В миссиях: брифинг — KORA, redTest — PIX,
  подсказки/разбор — KORA. Маппинг по существующему полю `speaker`/типу панели,
  схема контента не меняется.
- Один новый арт сцены: `scene-7-role` (экипаж + загорающийся спутник) для intro_navigator
  (сейчас там повтор scene-2). Остальной арт сцен НЕ перегенерируется (утверждён).
- Ограничение: codex-image требует VPN (OpenAI гео-блок) и делит квоту с codex CLI;
  при 429/гео — портреты откладываются, остальная пачка не блокируется.

## 3. Упрощение UI (утверждено ранее)

- Человеческие слова: вкладки «Играть / Карта навыков / Профиль»; «Курс: Основы кода»;
  «Миссия на сегодня · ~N мин»; кнопка «Играть»; статусы «не начата / уже знаешь — можно
  пропустить / пройдена / пора освежить»; узлы карты подписываются именем навыка
  (skillNames), код спутника мелко; уровни «ещё не изучено / узнал идею / умеешь читать /
  владеешь». Лор-термины остаются флейвором в подзаголовках.
- Прогрессивное раскрытие: после калибровки игрок попадает сразу в миссию дня; до первой
  завершённой миссии нижней навигации нет; после первой миссии появляется «Карта навыков»
  (реплика KORA «палуба вернулась в строй»), профиль — третьим. Экран «Играть»: одна
  карточка миссии дня, полный список свёрнут под «Все миссии главы ▸».
- Первое появление термина — краткая подпись обычными словами.

## 4. Моушн (библиотека `motion`, бесплатная; скиллы framer-motion-skills + design-motion-principles)

- Пролог: ken-burns/параллакс по арту, реплики построчно, переход сцен — направленный свайп.
- Переходы стадий и палуб — AnimatePresence (slide/fade).
- Задания: варианты каскадом (stagger), неверный ответ — shake + вспышка панели PIX,
  верный — pop; завершение миссии — вспышка/glow спутника.
- Комикс-SFX «БИП-БИП!» / «ОК!» / «ОНЛАЙН!»: CSS/SVG-бабблы с Motion-выскоком на событиях
  (ошибка — PIX «БИП-БИП!», верный ответ — «ОК!», миссия завершена — «СПУТНИК ОНЛАЙН!»).
  Текстовые, не картинки — легко править и не весят.
- `prefers-reduced-motion` уважается: анимации отключаются, смысл не теряется (канон).
- Бюджет: +~35KB gzip (motion); порог не критичен для PWA.
