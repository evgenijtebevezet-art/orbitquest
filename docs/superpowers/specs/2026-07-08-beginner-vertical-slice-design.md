# OrbitQuest — вертикальный срез для первого игрока-новичка

## Контекст и цель

Первый пользователь продукта — его создатель, **полный новичок в коде** («ноль понимания»).
Существующая база (пролог, персонажи, маршруты, MissionCore) сохраняется; меняется приоритет:
маршрут «Освоение» и контент с нулевым порогом входа строятся первыми.

Цель игрока на первый сезон: **научиться читать и проверять код** (Python) и понимать,
что реально происходит, когда AI пишет код за него. Не цель: писать код с нуля,
профессиональные ранги, подготовка к работе.

Форма: игра с сюжетом, героями и миссиями — не тест-тренажёр. Игрок проходит одну
миссию в день за 10–20 минут с телефона.

## Решения брейншторма (зафиксированы с владельцем)

| Вопрос | Решение |
|---|---|
| Чему учить | Два сектора по выбору игрока: «Основы кода» (Python) и «AI-кодинг» |
| Язык первого сезона | Python (читать/проверять; TypeScript — потенциальный сезон 2) |
| Мозг KORA | Гибрид: сценарные диалоги + живая кнопка «спросить KORA» через LLM, при недоступности — тихий фолбэк на заготовленные ответы |
| Где играть | Телефон, онлайн: PWA на GitHub Pages; ноутбук не нужен для игры |
| Ключи LLM | Cloudflare Worker (бесплатный tier), ключи NIM/Groq в секретах воркера; в клиенте ключей нет |
| Ритм | Миссия 10–20 минут, одна в день; напоминания в v1 нет |
| Анти-паттерны владельца | Без нудных повторов; каждая миссия отвечает «зачем»; взрослый тон без «молодец!» и стриков с виной |
| Арт-стиль | Взрослый 2D-мульт (референс Рик и Морти): жирный контур, плоские насыщенные цвета, кислотные акценты. Утверждён по сгенерированным сценам |
| Фоновая живность | Пришельцы-существа допустимы как визуальный юмор фона; говорящий ансамбль не меняется: Навигатор, KORA, PIX, VEGA |
| Связь с другими проектами владельца | Никакой: OrbitQuest самодостаточен, примеры кода собственные |

## Скоуп v1 (вертикальный срез)

Полный путь игрока, тонкий но цельный:

1. **Пролог** — 6 сцен по [[game/Prologue]]: сигнал → мостик → KORA → Atlas и Сдвиг → VEGA → PIX
   (+ сцена роли Навигатора текстом). Полноэкранный арт (уже сгенерирован,
   `assets/prologue-scenes/`), 2–3 реплики на сцену, одно действие игрока на сцену.
   Прогресс сцен сохраняется; после перезапуска пролог продолжается, а не начинается заново.
2. **Калибровка** — 6–8 заданий с честной кнопкой «Не знаю», детерминированный скоринг,
   результат: карта навыков + рекомендация маршрута с объяснением + ручной override.
3. **Выбор сектора** — «Основы кода» или «AI-кодинг»; переключение в любой момент,
   прогресс раздельный по секторам.
4. **Глава 1 «Основы кода»** — ~6 миссий (спутники CODE-01…CODE-06): что такое программа
   и буквальность машины; переменные и данные; порядок выполнения; условия; циклы;
   чтение целого маленького скрипта.
5. **Глава 1 «AI-кодинг»** — ~3 миссии (AGENT-01…AGENT-03): модель/инструкции/контекст;
   почему AI уверенно врёт и как проверять; инструменты и границы разрешений.
6. **Кнопка «Спросить KORA»** — в брифинге и после ошибки; ходит в kora-worker,
   фолбэк — заготовленные ответы миссии.
7. **Прогресс** — localStorage + экспорт/импорт JSON-файлом.

### Не входит в v1 (осознанно)

Pyodide/исполнение кода; Telegram-уведомления; звук; delta-миссии по реальным изменениям
технологий; облачная синхронизация; BKT/knowledge tracing (только прозрачные правила).
Каркас (схема контента, состояния миссии) закладывается так, чтобы это добавлялось без слома.

## Устройство миссии

Формат: **сигнал → брифинг → 3–5 заданий → доказательство → результат**.

- Брифинг ≤ 40 секунд чтения, всегда начинается с «зачем» (реальная жизнь, без лора в критериях).
- Типы заданий v1 (все проверяются детерминированно): выбрать верное объяснение;
  предсказать вывод кода; расставить шаги/строки по порядку; найти строку с ошибкой;
  сказать, что изменилось между двумя версиями кода (мини-diff).
- Ошибка → PIX показывает «красный тест» (факт), KORA — подсказки по нарастающей:
  1) наводящий вопрос → 2) направление → 3) разбор. Готовое решение до содержательной
  попытки не выдаётся. Подсказки фиксируются в attempt, не наказывают.
- Повторение: без повторов внутри миссии; навык возвращается через интервал в новой
  миссии в другом контексте (rule-based планировщик).
- Capability на навык: `unknown → recognizes → applies → transfers`; карта Atlas —
  визуализация этих состояний (спутник = навык).

## Контент как данные

- Миссии, сцены пролога и вопросы калибровки — файлы контента (JSON/MD) в `content/`
  по схеме из `packages/contracts`; код движка контента не знает.
- Схема валидируется в CI по всем файлам; сломанный контент не собирается.
- Диалоги пишутся по канону [[game/Characters]] (формулы речи KORA/PIX/VEGA), взрослый тон.

## Архитектура

```text
apps/web            React PWA (GitHub Pages, offline-first)
├── prologue        машина сцен (intro_* состояния из game/Prologue)
├── diagnostic      калибровка + детерминированный скоринг
├── missions        машина состояний миссии + рендеры типов заданий
├── atlas           карта созвездия (SVG/Canvas)
└── profile         localStorage + экспорт/импорт

apps/kora-worker    Cloudflare Worker: POST /ask
├── секреты: ключи NIM/Groq (в клиенте ключей нет)
├── системный промпт KORA (ограничения канона: не выдаёт решение, не судит код)
└── таймаут/ошибка → клиент молча берёт фолбэк-ответы из контента миссии

packages/contracts  типы + схема контента (+ валидатор)
content/            сцены, миссии, калибровка (данные)
assets/             сгенерированный арт (версионируется)
apps/api            остаётся для локальной разработки; в v1 не деплоится
```

- Деплой: GitHub Actions → GitHub Pages (сборка web + валидация контента + тесты).
- Игровой движок не используется (ADR-001 подтверждён): сцены/миссии — конечные автоматы,
  анимации — CSS/SVG.
- ADR-002 («только Gemini в runtime») **пересматривается**: runtime-LLM = NIM/Groq через
  kora-worker, т.к. Gemini недоступен владельцу (гео-блок, ключ suspended). Требуется ADR-004.

## Арт-пайплайн

- Генерация: скилл codex-image (gpt-image-2 по подписке Codex), промпт-шаблон с
  зафиксированными стилевыми токенами; готовые сцены — референсы для консистентности
  персонажей. Детерминированная графика (иконки, диаграммы, типографика) — руками SVG.
- Всё складывается в `assets/` и версионируется; в рантайме генерации нет.
- UI перекрашивается под стиль: тёмный фон, жирные контуры, кислотные акценты
  (текущий неон — временный).

## Тесты и проверка готовности

- Юниты: машина миссии, скоринг калибровки, планировщик повторений, валидатор контента.
- CI: typecheck + тесты + валидация всего `content/` + сборка + smoke.
- Приёмка v1 (все пункты проверяемы):
  - [ ] новый профиль не попадает на Мостик до пролога или явного skip;
  - [ ] пролог проходится за 3–4 минуты, каждая сцена переживает перезагрузку;
  - [ ] калибровка работает без сети и без LLM;
  - [ ] три fixture-профиля получают ожидаемые маршруты; override работает;
  - [ ] обе главы проходимы до конца на телефоне (390×844 и 360×800), без горизонтального скролла;
  - [ ] кнопка KORA: живой ответ при доступном воркере, фолбэк без ошибок при недоступном;
  - [ ] прогресс переживает reload и переустановку PWA через экспорт/импорт;
  - [ ] PWA ставится на Android-домашний экран с GitHub Pages;
  - [ ] tests, typecheck, build, контент-валидация зелёные в CI.

## Порядок реализации (крупно)

1. Контракты и схема контента + валидатор (+ ADR-004).
2. Машина пролога + 6 сцен контентом + арт подключён.
3. Калибровка + route reveal + выбор сектора.
4. Движок миссий + типы заданий + глава «Основы кода» (6 миссий).
5. Глава «AI-кодинг» (3 миссии) + карта Atlas с состояниями.
6. kora-worker + кнопка KORA + фолбэки.
7. GitHub Pages деплой + PWA-проверка на телефоне владельца.

Детализация — в implementation-плане (следующий документ).
