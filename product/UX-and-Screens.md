# UX и экраны

## Принцип

OrbitQuest ощущается местом, а не аналитической панелью. При этом пользователь сначала знакомится с миром и только затем получает постоянную навигацию.

Mobile-first цель: portrait `360–430px`, touch targets `44–48px`, safe areas, экранная клавиатура и reduced motion.

## Первый запуск

```text
Invitation → Ship Boot → KORA → Atlas/Shift
→ VEGA → PIX → Navigator Contract
→ Diagnostic → Route Reveal → Bridge
```

### 1. Invitation

- один входящий сигнал;
- нет постоянной навигации;
- CTA `Принять подключение`;
- работает без звука.

### 2. Ship Boot

- вертикальная сцена мостика;
- три последовательных переключателя: питание, связь, навигация;
- явно указано, что Odyssey — корабль;
- состояние сохраняется после каждого шага.

### 3. Character introductions

- KORA появляется после запуска навигации;
- VEGA появляется после демонстрации Сдвига;
- PIX появляется только в Ангаре;
- один персонаж говорит на сцене;
- текст доступен в виде субтитров;
- после пролога биографии доступны в «Экипаже».

### 4. Diagnostic

- 5–8 коротких интерактивных заданий;
- индикатор прогресса без оценочного ранга;
- кнопка `Не знаю`;
- confidence после ключевых ответов;
- code task открывается отдельным mobile editor state;
- диагностику можно прервать и продолжить.

### 5. Route Reveal

- карта сильных и неизвестных секторов;
- рекомендованный маршрут: Освоение, Практика или Актуализация;
- объяснение причин;
- возможность изменить выбор;
- CTA ведёт к первой подходящей миссии.

## Постоянные палубы

### Мостик

- текущая миссия;
- активный спутник;
- KORA и один контекстный вопрос;
- состояние Atlas;
- одна primary action;
- на mobile — вертикальная сцена и bottom navigation.

### Созвездие

- свободная карта навыков;
- capability и freshness визуально различаются;
- маршруты и prerequisites;
- объяснение рекомендации;
- ручной выбор сектора.

### Радар

- передачи VEGA;
- verified и unverified разделены;
- source, version, date, impact, confidence;
- создание delta-миссии только после approval.

### Ангар

- объект ремонта;
- mobile editor с ограниченной областью редактирования;
- PIX, test console и timeout state;
- подсказки слоями;
- diff попыток;
- реальные acceptance criteria.

### Архив

- claims и evidence;
- версии уроков;
- диагностические результаты;
- прошлые attempts;
- mastery/freshness history;
- возможность оспорить evaluator result.

### Экспедиции

- реальные проекты;
- несколько этапов;
- выбор инструментов;
- финальный артефакт;
- post-mission разбор.

### Экипаж

- KORA, PIX и VEGA после знакомства;
- профиль Навигатора;
- маршрут по секторам;
- повторная диагностика;
- настройка глубины KORA;
- приватность, экспорт и удаление данных.

## HUD после onboarding

Постоянно видны только:

- текущая миссия;
- состояние сигнала;
- доступ к KORA;
- bottom navigation;
- один индикатор стабильности.

В прологе HUD скрыт, чтобы пользователь не видел неизвестные палубы и персонажей до знакомства.

## Animation contract

- анимация показывает изменение состояния, а не украшает ожидание;
- интерфейс остаётся понятным при отключённой анимации;
- KORA, PIX и сигналы имеют состояния `idle`, `active`, `warning`, `success`;
- generated images загружаются как versioned assets;
- runtime image generation не входит в обычную миссию;
- тяжёлое 3D не блокирует MVP.
