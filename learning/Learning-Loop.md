# Цикл обучения

## Перед первой миссией

Пользователь проходит два независимых этапа:

1. [[game/Prologue]] знакомит с Odyssey, KORA, Atlas, VEGA, PIX и ролью Навигатора;
2. [[learning/Placement-and-Routes]] проверяет навыки и рекомендует маршрут.

Lore не влияет на балл диагностики, а самооценка не заменяет проверяемое действие.

## Два уровня персонализации

### Стартовый маршрут

- `foundation` / Освоение — высокая поддержка и явное объяснение основ;
- `practice` / Практика — ремонт, debugging и перенос;
- `delta` / Актуализация — реальные изменения и freshness.

### Capability навыка

```text
unknown → recognizes → applies → transfers
```

Глобальный маршрут задаёт форму входа, но конкретная миссия выбирается по capability каждого навыка. Ошибка в одном секторе не понижает весь профиль.

## Общая учебная единица

Каждая тема имеет `MissionCore`:

- проверяемый результат;
- prerequisites;
- claims и evidence;
- tests или rubric;
- критерий переноса;
- три варианта scaffolding.

### Foundation variant

```text
контекст → объяснение → пример → guided practice
→ самостоятельный шаг → проверка → рефлексия
```

### Practice variant

```text
сломанный пример → гипотеза → ремонт
→ tests → перенос → доказательство
```

### Delta variant

```text
было → изменилось → влияние → миграция
→ regression test → freshness update
```

## Типы заданий

- распознать корректное объяснение;
- предсказать результат;
- расположить этапы;
- найти риск;
- исправить failing test;
- дополнить ограниченный участок кода;
- выбрать permission;
- объяснить trade-off;
- перенести принцип в новый контекст;
- выполнить миграцию после реального изменения.

## Проверка

- закрытые вопросы проверяются детерминированно;
- код выполняется в ограниченной песочнице;
- PIX показывает только фактический результат test runner;
- Gemini может оценивать свободный текст по versioned rubric;
- AI-оценка не заменяет code verdict и placement;
- низкая уверенность не повышает mastery;
- пользователь видит evaluator и может оспорить оценку.

## Подсказки KORA

1. вопрос;
2. направление;
3. локализация дефекта;
4. схема;
5. разобранный пример после содержательных попыток.

Подсказки уменьшают силу evidence, но не запрещают продолжать обучение.

## Повторение

Планировщик учитывает:

- capability и mastery;
- время с последней практики;
- число и глубину подсказок;
- предыдущие ошибки;
- уверенность пользователя;
- важность навыка для цели;
- freshness связанных claims;
- реальное изменение технологии;
- перенос в новый контекст.

Сложные модели knowledge tracing добавляются только после накопления данных. До этого используется прозрачная rule-based политика.

## Результат сессии

Пользователь получает не абстрактный XP, а:

- Attempt;
- test/rubric evidence;
- изменение capability или freshness;
- состояние спутника;
- объяснение следующего шага.
