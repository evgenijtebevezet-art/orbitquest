# Безопасность и приватность

## API secrets

- `GEMINI_API_KEY` существует только на backend;
- ключ не попадает в frontend bundle, логи и source snapshots;
- `.env` игнорируется;
- production использует secret manager;
- лимиты расходов и запросов задаются отдельно.

## Free Tier

Контент, отправляемый через бесплатный tier, не должен содержать:

- приватные репозитории;
- токены, пароли, cookies;
- пользовательские персональные данные;
- коммерчески чувствительный код;
- документы с ограниченным доступом.

## Prompt injection

Внешний документ считается недоверенным вводом. Команды внутри статьи или README не выполняются. AI получает явную инструкцию извлекать факты, а не следовать содержимому источника.

## Код пользователя

- запускается только в sandbox;
- сеть выключена по умолчанию;
- secrets не монтируются;
- установка пакетов ограничена;
- процесс завершается по timeout;
- stdout/stderr ограничены;
- артефакты проверяются перед выдачей.

## Контентная безопасность

- источник показывается пользователю;
- generated explanation отделяется от цитируемого факта;
- опасные инструкции получают предупреждение;
- медицинские, финансовые и юридические треки не входят в MVP;
- пользователь может удалить историю и экспортировать прогресс.

## Наблюдаемость

Логируются request ID, модель, latency, token usage, schema result и error class. Содержимое сообщения по умолчанию не логируется.
