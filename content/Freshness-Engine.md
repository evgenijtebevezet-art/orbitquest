# Движок актуальности

## Pipeline

```text
Registry источников
  → HTTP/RSS/GitHub collectors
  → normalize + hash + dedupe
  → AI extraction
  → rule validation
  → impact analysis
  → draft claim/update
  → publish policy
  → affected lessons
  → delta mission
```

## Классы изменений

- `release` — новая доступная возможность;
- `behavior_change` — изменилось поведение;
- `deprecation` — объявлено прекращение;
- `removal` — возможность удалена;
- `security` — изменились риски или требования;
- `pricing_limit` — изменились стоимость или квоты;
- `workflow_shift` — появился более эффективный процесс;
- `research` — перспективный, но ещё не production-факт.

## Freshness TTL

| Тип знания | Повторная проверка |
|---|---:|
| модель, лимит, preview-функция | 7 дней |
| API/SDK поведение | 14 дней |
| инструмент разработки | 30 дней |
| стабильный framework | 90 дней |
| фундаментальный принцип | 365 дней |

TTL запускает проверку, но не означает автоматическое устаревание.

## Impact analysis

Для нового события система ищет:

1. прямые ссылки claim → source;
2. совпадающие product/version/entity;
3. семантически близкие claims;
4. уроки, использующие затронутые claims;
5. пользователей, изучавших эти уроки.

## Версионирование

Опубликованный урок не редактируется «на месте». Создаётся новая версия с diff и причиной. Старые ответы пользователя остаются привязаны к версии, на которой были получены.

## Конфликт источников

- Tier A побеждает Tier B только в пределах заявленной области;
- два противоречащих Tier A источника блокируют публикацию;
- claim получает `uncertain`;
- пользователю показывается, что вопрос ещё не разрешён.
