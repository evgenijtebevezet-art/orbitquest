# Стратегия Gemini API Free Tier

## Решение MVP

Runtime использует только Gemini Developer API. CLI-интеграций нет.

Предварительное распределение:

- `gemini-3.1-flash-lite` — классификация, extraction, простые адаптации и quiz;
- доступная Flash-модель повышенного качества — сложное объяснение и оценка;
- `gemini-embedding-2` — поиск по источникам, урокам и ошибкам;
- TTS Preview — только опциональная озвучка и только при доступной квоте.

Точный список моделей и лимитов читается из конфигурации окружения и проверяется при запуске. Preview-модель не является обязательной зависимостью.

## Экономия запросов

1. Источники загружаются без модели.
2. Хэш документа предотвращает повторный анализ.
3. Несколько мелких событий объединяются в один extraction batch.
4. Канонический урок генерируется один раз.
5. Персонализация меняет только нужные части.
6. Embeddings вычисляются один раз на версию документа.
7. Ответы на типовые вопросы кэшируются по смысловому ключу.

## Бюджет одного пользователя

- один фоновый пакет актуализации;
- одна ежедневная миссия;
- до трёх подсказок;
- одна оценка свободного ответа;
- embeddings только для нового контента.

При достижении лимита приложение продолжает работать: показывает подготовленные уроки, выполняет детерминированные задания и откладывает AI-оценку.

## Provider boundary

Бизнес-логика не импортирует SDK напрямую. Она зависит от интерфейса:

```ts
interface LearningAI {
  extractUpdate(input: SourceDocument): Promise<UpdateCandidate>;
  adaptLesson(input: LessonContext): Promise<AdaptedLesson>;
  evaluateExplanation(input: EvaluationInput): Promise<Evaluation>;
  embed(texts: string[]): Promise<number[][]>;
}
```

Это не означает мультипровайдерность MVP, но предотвращает связывание данных приложения с конкретным transport API.

## Privacy

Free Tier не используется для приватных репозиториев, секретов и персональных данных. Пользователю показывается явное предупреждение перед отправкой собственного текста или кода.
