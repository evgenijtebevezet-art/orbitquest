# Игровая система

OrbitQuest — serious game, где обучающее действие и действие внутри мира совпадают. Пользователь не получает награду за просмотр: мир меняется после проверяемого решения.

## Базовая фантазия

Пользователь — Навигатор Odyssey. Он обслуживает Atlas вместе с KORA, PIX и VEGA. Сдвиг превращает реальные изменения технологий и забывание навыков в состояние игрового мира.

Первый вход начинается с [[game/Prologue]], а не с dashboard или случайной миссии.

## Три режима игры

| Маршрут | Игровая роль | Главная деятельность | Поддержка |
|---|---|---|---|
| Освоение | оператор новых систем | активировать и понять | высокая |
| Практика | инженер восстановления | диагностировать и ремонтировать | адаптивная |
| Актуализация | аудитор Atlas | проверять diff и мигрировать | минимальная |

Маршрут не является постоянным рангом. Он может различаться по секторам и меняется на основе evidence. Подробно: [[learning/Placement-and-Routes]].

## Сопоставление механик

| Обучение | Игровой мир | Ведущая роль |
|---|---|---|
| диагностика | калибровка Навигатора | KORA |
| первичный источник | передача с Земли | VEGA |
| навык | спутник или модуль | Atlas |
| объяснение | навигационная схема | KORA |
| guided practice | калибровка | KORA + PIX |
| самостоятельная задача | ремонт | Навигатор |
| tests | симулятор | PIX |
| повторение | плановое обслуживание | Odyssey |
| устаревание | Сдвиг орбиты | мир |
| большой проект | экспедиция | экипаж |
| mastery evidence | стабильный канал | Atlas |

## Осмысленные награды

- восстановленный сектор;
- работающий объект корабля;
- подтверждённый спутник;
- новый маршрут;
- инструменты PIX, соответствующие реальной практике;
- новые визуализации KORA;
- архив реальных проектов и доказательств;
- фрагменты истории после содержательных действий.

## Не является наградой

- XP за открытие карточки;
- loot boxes;
- энергия, запрещающая продолжить обучение;
- наказание за пропуск дня;
- глобальный leaderboard;
- профессиональный ранг, основанный на кликах;
- косметика вместо образовательного результата.

## Failure state

Ошибка не уничтожает прогресс. PIX показывает детерминированный результат, KORA предлагает следующий слой scaffolding, а система сохраняет попытку как диагностическое evidence.

После повторных ошибок меняется не «здоровье игрока», а форма поддержки:

1. вопрос;
2. направление;
3. локализация дефекта;
4. схема;
5. разобранный пример;
6. новая самостоятельная задача.

## Мотивация

- **автономия:** маршрут объясняется и может быть изменён;
- **компетентность:** мир реагирует на доказанное действие;
- **отношения:** роли экипажа раскрываются через совместную работу;
- **любопытство:** новые сигналы и неизвестные сектора;
- **смысл:** объекты мира соответствуют реальным навыкам;
- **доверие:** источник, evaluator и ограничения видимы.

## Ограничения

- сюжет не заменяет ясное задание;
- сложность не выражается только количеством текста;
- персонажи не сообщают результат вне своей роли;
- обучение остаётся доступным без анимации;
- мобильный UX важнее декоративной сложности;
- live-service не создаёт искусственные инфоповоды.
