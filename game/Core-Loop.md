# Игровые циклы

## Цикл первого запуска

```text
Приглашение → Odyssey → KORA → Atlas/Сдвиг
→ VEGA → PIX → роль Навигатора
→ диагностика → маршрут → первая миссия
```

Полная режиссура: [[game/Prologue]].

## Общий цикл миссии

```text
Сигнал или цель → Верификация → Брифинг → Действие
→ Детерминированная проверка → Перенос → Evidence
→ Стабилизация → Повторение или новое изменение
```

### 1. Вход

- плановая foundation/practice-миссия выбирается по capability и цели;
- delta-миссия начинается только с подтверждённого изменения;
- если важного изменения нет, система не создаёт искусственный сигнал.

### 2. Верификация VEGA

Для изменяемого знания видимы источник, версия, дата, влияние и confidence. Foundation-миссия может использовать заранее утверждённый канонический материал.

### 3. Брифинг KORA

Одна цель, причина и критерий успеха. Глубина зависит от маршрута, но acceptance criteria не скрываются.

### 4. Действие Навигатора

Пользователь читает, прогнозирует, исправляет, пишет или сравнивает. До содержательной попытки KORA не показывает готовое решение.

### 5. Симулятор PIX

Код проверяется tests. Закрытые задания проверяются детерминированно. Свободное объяснение оценивается отдельно по rubric и не заменяет code verdict.

### 6. Перенос

Критический навык проверяется в отличающемся контексте. Успех только в знакомом примере не переводит capability в `transfers`.

### 7. Evidence и стабилизация

Attempt, evaluator, lesson version и test result сохраняются. Только после этого меняется состояние спутника.

## Варианты по маршрутам

### Освоение, 12–18 минут

```text
контекст → объяснение → пример → guided practice
→ маленькая самостоятельная задача → tests → рефлексия
```

### Практика, 10–15 минут

```text
дефект → гипотеза → самостоятельный ремонт
→ tests → вопрос на перенос → evidence
```

### Актуализация, 5–10 минут

```text
подтверждённый diff → влияние → миграция или аудит
→ regression tests → freshness update
```

## Повторение

Следующая миссия может быть:

- повторением в другом формате;
- переносом в новый контекст;
- устранением локального пробела;
- обновлением после реального изменения;
- частью сквозной экспедиции.

Маршрут пользователя не меняется из-за одной ошибки. Меняется уровень поддержки конкретного навыка.
